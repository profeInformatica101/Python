Infraestructura de Usuarios, Grupos y Permisos con AAA

En esta guía aprenderás a implementar una infraestructura de usuarios y permisos basada en el modelo AAA (Authentication, Authorization, Accounting) en un sistema Linux.

Modelo AAA

El modelo AAA se compone de tres elementos fundamentales:

  • Authentication (Autenticación): Identificación segura de los usuarios mediante nombres de usuario y contraseñas.
  • Authorization (Autorización): Control de acceso a recursos del sistema según los permisos asignados.
  • Accounting (Registro y Auditoría): Registro de las actividades de los usuarios para garantizar trazabilidad y seguridad.

1. Crear Usuarios

Ejecuta los siguientes comandos para crear los usuarios y asignarles contraseñas:

sudo useradd -m -s /bin/bash admin
echo "admin:admin1234" | sudo chpasswd

sudo useradd -m -s /bin/bash juan
echo "juan:juan1234" | sudo chpasswd

sudo useradd -m -s /bin/bash maria
echo "maria:maria1234" | sudo chpasswd

sudo useradd -m -s /bin/bash lucas
echo "lucas:lucas1234" | sudo chpasswd

Relación con AAA: Estos comandos implementan la autenticación al crear usuarios únicos con credenciales seguras almacenadas en el sistema.

2. Crear Grupos

Define los grupos necesarios y asigna usuarios a estos grupos:

sudo groupadd administradores
sudo usermod -aG administradores admin

sudo groupadd desarrollo
sudo usermod -aG desarrollo juan
sudo usermod -aG desarrollo maria

sudo groupadd testing
sudo usermod -aG testing lucas

Relación con AAA: La autorización se aplica al definir qué grupos tienen acceso a recursos específicos en el sistema.

3. Crear Directorios y Permisos

Crea los directorios y establece permisos específicos según los grupos:

/home/admin

sudo mkdir -p /home/admin
sudo chown admin:administradores /home/admin
sudo chmod 700 /home/admin

Relación con AAA: El directorio personal de cada usuario tiene permisos exclusivos, asegurando que solo el propietario pueda acceder.

/proyectos

sudo mkdir -p /proyectos
sudo chown admin:desarrollo /proyectos
sudo chmod 770 /proyectos
sudo setfacl -m g:testing:rx /proyectos

Relación con AAA: Los permisos de lectura/escritura para el grupo desarrollo y de solo lectura para testing implementan la autorización.

/testing

sudo mkdir -p /testing
sudo chown lucas:testing /testing
sudo chmod 770 /testing
sudo setfacl -m g:administradores:rx /testing

Relación con AAA: La autorización asegura que solo los usuarios necesarios accedan a los recursos.

4. Habilitar Auditoría

Registra las actividades de los usuarios en el sistema:

sudo apt install auditd -y
sudo auditctl -a always,exit -F dir=/proyectos -F perm=rwx -k proyectos_audit
sudo auditctl -a always,exit -F dir=/testing -F perm=rwx -k testing_audit

Consulta los logs de auditoría con el comando:

sudo ausearch -k proyectos_audit
sudo ausearch -k testing_audit

Relación con AAA: El registro y auditoría aseguran que todas las actividades en los directorios monitoreados sean rastreables.

5. Resumen

En este ejemplo:

  • Autenticación: Los usuarios se crean con credenciales únicas.
  • Autorización: Se definen permisos específicos para los usuarios y grupos en los directorios.
  • Registro y Auditoría: Se habilita auditoría en los directorios críticos para rastrear accesos y modificaciones.
